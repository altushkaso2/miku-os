## О проекте

**Miku OS** - UNIX-подобная операционная система, разработанная с нуля в среде `no_std`.
Не использует стандартную библиотеку (`libc`), полный контроль над железом и архитектурой памяти.

> Весь код написан на Rust. Ассемблер используется только в загрузчике, обработчике syscall и переключении контекста.

## Технические характеристики

### Ядро

| Компонент | Описание |
|:--|:--|
| **Архитектура** | x86_64, `#![no_std]`, `#![no_main]` |

| **Syscall** | SYSCALL/SYSRET через MSR, naked asm обработчик |

### Управление памятью

<details>
<summary><b>Физическая память (PMM)</b></summary>

#### Аллокатор фреймов

- **Bitmap аллокатор** - до 4M фреймов (16 ГБ RAM), 1 бит = 1 фрейм 4KB
- **free_hint** и **contiguous_hint** - ускоряют поиск свободных и непрерывных фреймов
- **Непрерывный alloc** - выделение N фреймов за один запрос
- **Регионы** - динамическая регистрация диапазонов RAM из карты памяти Multiboot2

#### Аварийный пул

Резерв фреймов для swap-in внутри обработчика page fault:

| Параметр | Значение |
|:--|:--|
| **Размер пула** | 64 фрейма (256 КБ) |
| **Назначение** | Только swap-in внутри page fault handler |
| **Пополнение** | Timer ISR вызывает `refill_emergency_pool_tick()` при 250 Гц |
| **Причина** | Обычный evict_one() вызывает ATA I/O - нельзя использовать в fault handler |

```
alloc_frame()           - обычный alloc из PMM
alloc_frame_emergency() - только аварийный пул (для fault handler)
alloc_or_evict()        - alloc + evict при нехватке RAM
alloc_for_swapin()      - только аварийный пул (fault context)
```

</details>

<details>
<summary><b>Виртуальная память (VMM)</b></summary>

- **4-уровневые таблицы страниц** (PML4 → PDP → PD → PT)
- **HHDM** - Higher Half Direct Map для доступа ядра к физической памяти (`0xFFFF800000000000`)
- **mark_swapped()** - запись swap PTE при выгрузке страницы
- Поддержка маппинга ring 0 / ring 3

</details>

<details>
<summary><b>Своп (Swap)</b></summary>

Полная реализация свопа на блочном устройстве (ATA диск):

#### Обратное отображение (swap_map)

- Для каждого физического фрейма хранится `(cr3, virt_addr, age, pinned)`
- Отслеживает до 512K фреймов (2 ГБ RAM)

#### Алгоритм выгрузки - Clock Sweep

```
Проход 1: ищем фреймы с age >= 3 (самые старые)
Проход 2: аварийный - берём любой unpinned фрейм
```

- `touch(phys)` - сбрасывает age в 1 при доступе к странице
- `age_all()` - увеличивает age всех фреймов по таймеру

#### Кодирование Swap PTE

```
бит 0     = 0  (PRESENT=0 - страница не в памяти)
бит 1     = 1  (SWAP_MARKER - отличает от unmapped)
биты 12.. = номер swap слота
```

#### Процесс выгрузки

```
evict_one():
  1. pick_victim() из swap_map + устанавливаем pinned=true
  2. swap_out_internal() → пишем страницу на диск
  3. vmm::mark_swapped() → обновляем PTE
  4. swap_map::untrack() → удаляем из обратной карты
  5. pmm::free_frame() → возвращаем фрейм
```

</details>

### Планировщик

| Параметр | Значение |
|:--|:--|
| **Алгоритм** | CFS (Completely Fair Scheduler), вытесняющий |
| **Максимум процессов** | 4096 |
| **Частота таймера** | 250 Гц (PIT) |
| **Окно CPU** | 250 тиков (1 секунда) |
| **Стек** | 512 КБ на процесс |
| **Состояния** | `Ready`, `Running`, `Sleeping`, `Blocked`, `Dead` |
| **Реализация** | Lock-free - ISR использует только атомики, без мьютексов |

Переключение контекста реализовано на naked asm. `schedule_from_isr` не берёт ни одного мьютекса.

### Системные вызовы

Реализованы через `SYSCALL/SYSRET` (MSR), naked asm обработчик с переключением стека через `swapgs`.

| Nr | Имя | Описание |
|:--:|:--|:--|
| **0** | `sys_write` | Запись в stdout/stderr (fd 1/2), до 4096 байт |
| **1** | `sys_read` | Чтение (заглушка) |
| **2** | `sys_exit` | Завершение процесса + yield |
| **3** | `sys_sleep` | Сон N тиков |
| **4** | `sys_getpid` | Получить PID текущего процесса |

### Сетевой стек

Полный сетевой стек реализован с нуля без сторонних библиотек.

<details>
<summary><b>Драйверы сетевых карт</b></summary>

| Драйвер | Чип |
|:--|:--|
| **Intel E1000** | 82540EM, 82545EM, 82574L, 82579LM, I217 |
| **Realtek RTL8139** | RTL8139 |
| **Realtek RTL8168** | RTL8168, RTL8169 |
| **VirtIO Net** | Виртуальная сетевая карта QEMU/KVM |

Все драйверы определяются автоматически через PCI сканер.

</details>

<details>
<summary><b>Протоколы</b></summary>

| Уровень | Протокол |
|:--|:--|
| **L2** | Ethernet, ARP (с таблицей кэша) |
| **L3** | IPv4, ICMP |
| **L4** | UDP, TCP (с управлением состоянием соединений) |
| **Приложение** | DHCP, DNS, NTP, HTTP, HTTP/2, Traceroute |
| **Безопасность** | TLS 1.3 (ECDHE + RSA + AES-GCM) |

</details>

<details>
<summary><b>TLS 1.3 - полная реализация с нуля</b></summary>

- **ECDH** - обмен ключами X25519 (`tls_ecdh.rs`)
- **RSA** - парсинг сертификатов ASN.1/DER, проверка подписи PKCS#1 (`tls_rsa.rs`)
- **BigNum** - собственная реализация длинной арифметики для RSA 2048-bit (`tls_bignum.rs`)
- **AES-GCM** - аутентифицированное симметричное шифрование (`tls_gcm.rs`)
- **SHA-256, HMAC, HKDF** - хэширование, выработка ключей (`tls_crypto.rs`)
- **Хэндшейк** - ClientHello → ServerHello → Certificate → Finished (`tls.rs`)

Без внешних крейтов, полностью в `no_std` окружении.

</details>

### VFS (Виртуальная файловая система)

<details>
<summary><b>Развернуть</b></summary>

#### Базовые параметры

| Параметр | Значение |
|:--|:--|
| **VNode** | 256 |
| **Открытых файлов одновременно** | 32 |
| **Точек монтирования** | 8 |
| **Дочерних узлов на директорию** | 32 |

- **Типы узлов**: `Regular`, `Directory`, `Symlink`, `CharDevice`, `BlockDevice`, `Pipe`, `Fifo`, `Socket`
- Полные метаданные: права, uid/gid, временные метки, размер, nlinks

#### Кэши

| Кэш | Размер |
|:--|:--|
| **Page cache** | 128 страниц x 512 байт, LRU вытеснение |
| **Dentry cache** | 128 записей, FNV32 хэш |

#### Навигация

- **Path walking** - глубина до 32 компонентов
- **Разрешение симлинков** - защита от циклов (8 уровней)
- **FNV32 хэш** - хэширование имён для O(1) поиска

#### Безопасность

- UNIX модель прав: `owner/group/other`, `setuid/setgid/sticky`
- Метки безопасности (MAC), квоты в байтах и inodes
- Блокировки файлов: shared/exclusive с обнаружением дедлоков (до 16 блокировок)

#### Расширенные возможности

| Функция | Детали |
|:--|:--|
| **VFS журнал** | 16 записей операций |
| **Xattr** | 8 расширенных атрибутов на узел |
| **Notify события** | inotify-подобная подсистема (до 16 событий) |
| **Version store** | 16 снапшотов файла |
| **CAS store** | Дедупликация по содержимому (до 16 объектов) |
| **Block I/O очередь** | 8 асинхронных запросов |

</details>

### MikuFS - Драйвер Ext2/3/4

<details>
<summary><b>Развернуть</b></summary>

#### Чтение

- Суперблок, групповые дескрипторы, inode, записи директорий
- Косвенные блоки (одинарные / двойные / тройные)
- Ext4 extent tree

#### Запись

- Создание и удаление файлов, директорий, символических ссылок
- Bitmap аллокатор для блоков и inode (с поддержкой предпочтительной группы)
- Рекурсивное удаление

#### Журнал Ext3 (JBD2)

- Создание журнала (конвертация `ext2 → ext3`)
- Запись транзакций: descriptor block, commit block, revoke block
- Recovery - при монтировании воспроизводит незавершённые транзакции

#### Утилиты

- `fsck` - проверка целостности
- `tree` - визуализация дерева директорий
- `du`, `cp`, `mv`, `chmod`, `chown`, жёсткие ссылки

</details>

### Консоль и фреймбуфер

<details>
<summary><b>Развернуть</b></summary>

Рендеринг реализован полностью вручную, без графических библиотек:

- **Двойной рендеринг** - кастомные bitmap глифы 9x16 + noto-sans-mono фоллбэк
- **Теневой буфер** - построчный u32 буфер для ускорения blit (bpp=4)

- **Скролл** - memmove пиксельных строк + очистка последней строки
- **RGB на символ** - каждая ячейка хранит `(ch, r, g, b)` независимо
- **Курсор** - вертикальный курсор шириной 2 пикселя с кастомным цветом

</details>

## Сборка и запуск

### Необходимые инструменты

| Инструмент | Назначение |
|:--|:--|
| **Rust nightly** | `no_std` + нестабильные возможности компилятора |
| **QEMU** | Эмуляция x86_64 машины |
| **grub-mkrescue** | Создание загрузочного ISO |
| **Cargo** | Сборка builder и ядра |

### Запуск

```bash
git clone https://github.com/altushkaso2/miku-os
cd miku-os/builder
cargo run
```

Builder делает всё автоматически:

```
Режим экономии RAM? (y/N)
[1/5] Компиляция ядра miku-os
[2/5] Создание файловой структуры (ввод размера диска и свопа)
[3/5] Генерация образа системы (miku-os.iso)
[4/5] Подготовка дисков
[5/5] Запуск QEMU (опционально (y/N))
```

> Первая сборка занимает несколько минут на загрузку зависимостей и компиляцию ядра.
> Последующие сборки занимают несколько секунд.
