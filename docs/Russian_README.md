## О проекте

**Miku OS** - UNIX-подобная операционная система, написанная с нуля в окружении `no_std`.
Не использует стандартную библиотеку (`libc`) и обеспечивает полный контроль над железом и архитектурой памяти.
ELF-динамическое связывание, разделяемые библиотеки и пользовательские процессы реализованы с нуля.

> Весь код написан на Rust. Ассемблер используется только в загрузчике, обработчике syscall и переключении контекста.

<details>
<summary><b>ld-miku (динамический компоновщик)</b></summary>

#### Общее описание

`ld-miku` - ELF-динамический компоновщик для MikuOS. Написан на Rust в окружении `#![no_std]`,
компилируется как статический PIE-бинарь.

#### Процесс работы

```
1. Ядро загружает ELF, обнаруживает PT_INTERP
2. ld-miku.so отображается в память из INCLUDE_BYTES
3. ld-miku запускается, разбирает AT_PHDR/AT_ENTRY из auxv
4. Определяет нужные библиотеки из DT_NEEDED
5. Отображает разделяемые библиотеки через syscall SYS_MAP_LIB
6. Применяет PLT/GOT релокации
7. Экспортирует символы в глобальную таблицу
8. Выполняет DT_INIT / DT_INIT_ARRAY
9. Переходит на точку входа исполняемого файла
```

#### Особенности

- Глобальная таблица символов (до 1024 символов)
- Разрешение weak-символов
- Рекурсивная загрузка зависимостей (до 16 библиотек)
- Поддержка релокации R_X86_64_COPY
- Корректный разбор auxv с пропуском envp

</details>

<details>
<summary><b>Разделяемые библиотеки (solib)</b></summary>

#### Глобальный кэш библиотек

| Параметр | Значение |
|:--|:--|
| **Макс. кэша** | 32 библиотеки |
| **Пути поиска** | /lib, /usr/lib |
| **Разделяемые страницы** | .text/.rodata - одни физические страницы для всех процессов |
| **Приватные страницы** | .data/.bss - копия для каждого процесса |

#### Syscall SYS_MAP_LIB (nr=15)

Ядро разбирает ELF-сегменты и отображает разделяемую библиотеку напрямую в адресное пространство процесса.

- read-only сегменты - разделяемые физические страницы (одинаковые для всех процессов)
- writable сегменты - новое выделение для каждого процесса

```
Процесс A: libmiku.so .text -> физ. страница 0x1234000 (общая)
Процесс B: libmiku.so .text -> физ. страница 0x1234000 (та же!)
Процесс A: libmiku.so .data -> физ. страница 0x5678000 (приватная)
Процесс B: libmiku.so .data -> физ. страница 0x9ABC000 (приватная)
```

#### Системная библиотека

`/lib/libmiku.so` встроена в VFS (tmpfs) как immutable-файл.
Флаг immutable запрещает unlink / write / rename.

| `ldconfig` | Сканировать /lib и /usr/lib, обновить кэш |
| `ldd` | Показать список закэшированных библиотек |

</details>

### Управление памятью

<details>
<summary><b>Физическая память (PMM)</b></summary>

#### Аллокатор фреймов

- Bitmap-аллокатор: до 4M фреймов (16 GB RAM), 1 бит = 1 фрейм 4 KB
- `free_hint` и `contiguous_hint` для ускорения поиска свободных фреймов
- Непрерывный alloc: резервирование N фреймов за один запрос
- Регионы: динамическая регистрация диапазонов RAM из Multiboot2 memory map

#### Аварийный пул

| Параметр | Значение |
|:--|:--|
| **Размер пула** | 64 фрейма (256 KB) |
| **Назначение** | Только для swap-in в обработчике page fault |
| **Пополнение** | Timer ISR вызывает `refill_emergency_pool_tick()` на частоте 250 Hz |

```
alloc_frame()           - обычный alloc из PMM
alloc_frame_emergency() - только аварийный пул (для fault-обработчика)
alloc_or_evict()        - alloc + вытеснение страницы при нехватке RAM
alloc_for_swapin()      - только аварийный пул (fault-контекст)
```

</details>

<details>
<summary><b>Виртуальная память (VMM)</b></summary>

- 4-уровневые таблицы страниц (PML4 -> PDP -> PD -> PT)
- HHDM: Higher Half Direct Map (`0xFFFF800000000000`)
- `mark_swapped()`: запись swap PTE при выгрузке страницы
- Поддержка маппинга ring 0 / ring 3
- Создание и уничтожение адресных пространств для пользовательских процессов

</details>

<details>
<summary><b>Подсистема mmap</b></summary>

| Параметр | Значение |
|:--|:--|
| **Диапазон MMAP** | 0x100000000 ~ 0x7F0000000000 |
| **Диапазон BRK** | 0x6000000000 ~ |
| **Макс. VMA** | 64 записи |
| **Функции** | mmap, munmap, mprotect, brk |

</details>

<details>
<summary><b>Swap</b></summary>

#### Обратное отображение (swap_map)

- Для каждого физического фрейма хранится `(cr3, virt_addr, age, pinned)`
- Отслеживает до 512K фреймов (2 GB RAM)

#### Алгоритм вытеснения: Clock Sweep

```
Pass 1: ищем фреймы с age >= 3 (самые старые)
Pass 2: аварийный режим - берём любой unpinned фрейм
```

- `touch(phys)`: сбрасывает age в 1 при обращении к странице
- `age_all()`: увеличивает age всех фреймов по таймеру

#### Кодировка Swap PTE

```
bit 0     = 0  (PRESENT=0)
bit 1     = 1  (SWAP_MARKER)
bits 12.. = номер swap-слота
```

</details>

### VFS (виртуальная файловая система)

<details>
<summary><b>Развернуть</b></summary>

#### Основные параметры

| Параметр | Значение |
|:--|:--|
| **Кол-во VNode** | 256 |
| **Одновременно открытых файлов** | 32 |
| **Точек монтирования** | 8 |
| **Дочерних узлов** | Динамически (без ограничений) |

Дочерние узлы управляются динамической хэш-картой на базе `Vec`. Начальный размер - 16 слотов, при достижении 75% заполненности автоматически увеличивается вдвое.

- Типы узлов: `Regular`, `Directory`, `Symlink`, `CharDevice`, `BlockDevice`, `Pipe`, `Fifo`, `Socket`
- Полные метаданные: права доступа, uid/gid, временные метки, размер, nlinks

#### Системная библиотека

При загрузке создаётся каталог `/lib` в tmpfs, в него записывается `libmiku.so` как immutable-файл.
Флаг immutable запрещает unlink / write / rename.

#### Кэши

| Кэш | Размер |
|:--|:--|
| **Page cache** | 128 страниц x 512 байт, вытеснение по LRU |
| **Dentry cache** | 128 записей, хэш FNV32 |

#### Навигация

- Path walking: глубина до 32 компонентов
- Разрешение символических ссылок: защита от циклов (8 уровней)
- FNV32-хэш: хэширование имён для O(1) поиска

#### Безопасность

- UNIX-модель прав: `owner/group/other`, `setuid/setgid/sticky`
- Метки безопасности (MAC), квоты по байтам и inode
- Блокировки файлов: shared/exclusive с обнаружением дедлоков (до 16 блокировок)
- Immutable-флаг: защита системных библиотек

#### Расширенные возможности

| Функция | Подробности |
|:--|:--|
| **VFS Journal** | Лог из 16 операций |
| **Xattr** | 8 расширенных атрибутов на узел |
| **Notify-события** | inotify-подобная подсистема (до 16 событий) |
| **Version store** | 16 снапшотов файла |
| **CAS store** | Дедупликация по содержимому (до 16 объектов) |
| **Block I/O queue** | 8 асинхронных запросов |

</details>

### MikuFS: драйвер Ext2/3/4

<details>
<summary><b>Развернуть</b></summary>

#### Чтение

- Суперблок, group descriptor, inode, записи каталогов
- Косвенные блоки (одинарные / двойные / тройные)
- Extent-дерево Ext4

#### Запись

- Создание и удаление файлов, каталогов, символических ссылок
- Bitmap-аллокатор блоков и inode (с поддержкой preferred group)
- Рекурсивное удаление
- Отложенная запись (dirty cache + pdflush)

#### Журнал Ext3 (JBD2)

- Создание журнала (конвертация `ext2 -> ext3`)
- Запись транзакций: descriptor block, commit block, revoke block
- Восстановление: replay незавершённых транзакций при монтировании
- Отложенный коммит: ускорение записи журнала через dirty cache

#### mkfs

- Форматирование ext2/ext3/ext4
- Lazy init: только метаданные группы 0 инициализируются сразу, остальное - отложенно
- Инициализация только journal superblock (обнуление всех блоков не требуется)

#### Утилиты

- `fsck`, `tree`, `du`, `cp`, `mv`, `chmod`, `chown`, hard link

</details>

### Userspace

<details>
<summary><b>Запуск процессов</b></summary>

#### Процесс запуска ELF

```
1. exec("test_dynamic")
2. Ядро: читает файл из ext2
3. Ядро: проверяет ELF-заголовок (magic, class, machine)
4. Ядро: отображает PT_LOAD сегменты в адресное пространство пользователя
5. Ядро: обнаруживает PT_INTERP, загружает ld-miku.so
6. Ядро: строит стек (argc, argv, envp, auxv)
7. Ядро: переходит на точку входа ld-miku (ring 3)
8. ld-miku: загружает библиотеки DT_NEEDED через SYS_MAP_LIB
9. ld-miku: применяет PLT/GOT релокации
10. ld-miku: переходит на _start исполняемого файла
```

| `mv <old> <new>` | Переименовать |
| `stat <path>` | Информация о файле |
| `chmod <mode> <path>` | Изменить права доступа |
| `df` | Информация о файловой системе |

#### Команды динамического связывания

| Команда | Описание |
|:--|:--|
| `exec <path>` | Запустить ELF-бинарь (с поддержкой динамического связывания) |
| `ldconfig` | Обновить кэш разделяемых библиотек |
| `ldd` | Показать список закэшированных библиотек |

#### Команды mkfs

| Команда | Описание |
|:--|:--|
| `mkfs.ext2 <drive>` | Форматировать в ext2 |
| `mkfs.ext3 <drive>` | Форматировать в ext3 (с журналом) |
| `mkfs.ext4 <drive>` | Форматировать в ext4 (extents + журнал) |

### ATA-драйвер

| Параметр | Значение |
|:--|:--|
| **Режим** | PIO (программный ввод/вывод) |
| **Операции** | Чтение/запись секторов (512 байт), до 255 секторов за команду |
| **Кол-во дисков** | 4: Primary/Secondary x Master/Slave |
| **Защита** | Сброс кэша после записи, таймаут 50K итераций |
| **Адресация** | LBA28 (до 128 GB) |
