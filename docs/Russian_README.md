## О проекте

**Miku OS** - это UNIX-подобная операционная система, написанная с нуля в окружении `no_std`.
Стандартная библиотека (`libc`) не используется вообще - полный контроль над железом и архитектурой памяти.

> Весь код написан на Rust. Ассемблер используется только в загрузчике, обработчике системных вызовов и переключении контекста.

## Технические характеристики

### Ядро

| Компонент | Описание |
|:--|:--|

| **PIC** | PIC8259 (смещения 32/40) |
| **Куча** | 128 МБ, аллокатор на связном списке |
| **Syscall** | SYSCALL/SYSRET через MSR, naked asm обработчик |

### Управление памятью

<details>
<summary><b>Физическая память (PMM)</b></summary>

#### Аллокатор фреймов

- Битмап-аллокатор - до 4M фреймов (16 ГБ RAM), 1 бит = 1 фрейм 4 КБ
- `free_hint` и `contiguous_hint` - быстрый поиск свободных и непрерывных фреймов
- Contiguous alloc - резервирование N фреймов за один запрос
- Регионы - динамическая регистрация диапазонов RAM из карты памяти Multiboot2

#### Аварийный пул

| Параметр | Значение |
|:--|:--|
| **Размер пула** | 64 фрейма (256 КБ) |
| **Назначение** | swap-in только внутри обработчика page fault |
| **Пополнение** | Timer ISR вызывает `refill_emergency_pool_tick()` каждые 250 Гц |

```
alloc_frame()           - обычный alloc из PMM
alloc_frame_emergency() - только аварийный пул (обработчик fault)
alloc_or_evict()        - alloc + вытеснение при нехватке RAM
alloc_for_swapin()      - только аварийный пул (контекст fault)
```

</details>

<details>
<summary><b>Виртуальная память (VMM)</b></summary>

- 4-уровневые таблицы страниц (PML4 -> PDP -> PD -> PT)
- HHDM - Higher Half Direct Map (`0xFFFF800000000000`)
- `mark_swapped()` - запись swap PTE при выгрузке страницы
- Поддержка маппинга ring 0 / ring 3

</details>

<details>
<summary><b>Своп (Swap)</b></summary>

#### Обратное отображение (swap_map)

- Для каждого физического фрейма хранится `(cr3, virt_addr, age, pinned)`
- Отслеживает до 512K фреймов (2 ГБ RAM)

#### Алгоритм вытеснения - Clock Sweep

```
Pass 1: ищем фреймы с age >= 3 (самые старые)
Pass 2: экстренный режим - берём любой unpinned фрейм
```

- `touch(phys)` - сбрасывает age в 1 при обращении к странице
- `age_all()` - увеличивает age всех фреймов по таймеру

#### Кодирование Swap PTE

```
bit 0     = 0  (PRESENT=0)
bit 1     = 1  (SWAP_MARKER)
bits 12.. = номер слота свопа
```

</details>

### Планировщик

| Параметр | Значение |
|:--|:--|
| **Тип** | CFS (Completely Fair Scheduler), вытесняющий |
| **Макс. процессов** | 4096 |
| **Частота таймера** | 250 Гц (PIT) |
| **Окно CPU** | 250 тиков (1 секунда) |
| **Стек** | 512 КБ на процесс |
| **Состояния** | Ready / Running / Sleeping / Blocked / Dead |
| **Реализация** | Lock-free - ISR использует только атомики, без мьютексов |

Переключение контекста реализовано на naked asm. `schedule_from_isr` не захватывает ни одного мьютекса.

<details>
<summary><b>TLS 1.3 - полная реализация с нуля</b></summary>

- ECDH - обмен ключами X25519 (`tls_ecdh.rs`)
- RSA - парсинг сертификатов ASN.1/DER, верификация подписи PKCS#1 (`tls_rsa.rs`)
- BigNum - собственная арифметика больших чисел для RSA 2048-bit (`tls_bignum.rs`)
- AES-GCM - аутентифицированное симметричное шифрование (`tls_gcm.rs`)
- SHA-256, HMAC, HKDF - хеширование, выведение ключей (`tls_crypto.rs`)
- Рукопожатие - ClientHello -> ServerHello -> Certificate -> Finished

Без внешних крейтов, реализовано с нуля в `no_std`.

</details>

### VFS (Виртуальная файловая система)

<details>
<summary><b>Развернуть</b></summary>

#### Основные параметры

| Параметр | Значение |
|:--|:--|
| **VNodes** | 256 |
| **Открытых файлов** | 32 одновременно |
| **Точек монтирования** | 8 |
| **Дочерних нод** | Динамически (без ограничений) |

Дочерние ноды хранятся в динамической хэш-карте на основе `Vec`. Начальная ёмкость - 16 слотов, при достижении 75% загрузки автоматически удваивается. Старое фиксированное ограничение в 32 дочерних ноды на директорию упразднено.

- Типы нод: `Regular`, `Directory`, `Symlink`, `CharDevice`, `BlockDevice`, `Pipe`, `Fifo`, `Socket`
- Полные метаданные: права, uid/gid, временные метки, размер, nlinks

#### Кэш

| Кэш | Размер |
|:--|:--|
| **Page cache** | 128 страниц x 512 байт, LRU вытеснение |
| **Dentry cache** | 128 записей, FNV32 хэш |

#### Навигация

- Path walking - глубина до 32 компонентов
- Разрешение симлинков - защита от петель (8 уровней)
- FNV32 хэш - поиск имён за O(1)

#### Безопасность

- UNIX модель прав: `owner/group/other`, `setuid/setgid/sticky`
- Метки безопасности (MAC), квоты в байтах и inodes
- Блокировки файлов: shared/exclusive с определением дедлока (макс. 16 блокировок)

#### Расширенные функции

| Функция | Детали |
|:--|:--|
| **VFS-журнал** | 16 записей операций |
| **Xattr** | 8 расширенных атрибутов на ноду |
| **Notify-события** | inotify-подобная подсистема (макс. 16 событий) |
| **Version store** | 16 снимков файла |
| **CAS store** | контентно-адресуемая дедупликация (макс. 16 объектов) |
| **Block I/O queue** | 8 асинхронных запросов |

</details>

### MikuFS - Драйвер Ext2/3/4

<details>
<summary><b>Развернуть</b></summary>

#### Чтение

- Суперблок, дескрипторы групп, inode, записи директорий
- Косвенные блоки (одинарные / двойные / тройные)
- Дерево экстентов Ext4

#### Запись

- Создание и удаление файлов, директорий, симлинков
- Битмап-аллокатор для блоков и inodes (с предпочтительной группой)
- Рекурсивное удаление

#### Журнал Ext3 (JBD2)

- Создание журнала (конвертация `ext2 -> ext3`)
- Запись транзакций: блок дескриптора, блок коммита, блок revoke
- Восстановление - воспроизведение незавершённых транзакций при монтировании

#### Утилиты

- `fsck`, `tree`, `du`, `cp`, `mv`, `chmod`, `chown`, жёсткие ссылки

</details>

## Сборка и запуск

### Необходимые инструменты

| Инструмент | Назначение |
|:--|:--|
| **Rust nightly** | `no_std` + нестабильные функции компилятора |
| **QEMU** | Эмуляция x86_64 |
| **grub-mkrescue** | Создание загрузочного ISO |
| **Cargo** | Сборка ядра |

### Шаги

```bash
git clone https://github.com/altushkaso2/miku-os
cd miku-os/builder
cargo run
```

Builder делает всё автоматически:

```
Режим экономии RAM? (y/N)
[1/5] Компиляция ядра miku-os
[2/5] Создание файловой структуры (введите размер диска и размер свопа)
[3/5] Генерация образа системы (miku-os.iso)
[4/5] Подготовка диска
[5/5] Запуск QEMU (опционально (y/N))
```

> Первая сборка занимает несколько минут на загрузку зависимостей и компиляцию ядра.
> Последующие сборки завершаются за секунды.
